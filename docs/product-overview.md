**risely（ライズリー）**

### GitHubリポジトリ名

## プロダクトの一言定義

| 項目    | 内容                                                                                             |
| ----- | ---------------------------------------------------------------------------------------------- |
| サービス名 | **risely（ライズリー）**                                                                    |
| 一言定義  | ウェアラブル・カレンダー・スマホのデータから、**「なぜ疲れるのか」「いつ集中できるのか」**という"あなた固有の生活パターン"を解析し、**毎朝「今日の1つだけの行動」を提案するAI** |
| カテゴリ  | Personal Health & Performance Intelligence（ヘルス×生産性コーチ）                                         |

## 想定ターゲットペルソナ

| 項目     | 内容                                                                                                                                               |
| ------ | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| 主ターゲット | 20–40代の知的労働者（PM / エンジニア / コンサル / スタートアップ系 / 医師など）                                                                                                |
| 条件     | すでに Oura / Apple Watch / Fitbit などを使っている or 睡眠・疲労に悩んでいる                                                                                          |

## risely が提供する価値（Value Proposition）

### コアメッセージ

> **"あなた自身がまだ気づいていない生活パターンを解析し、

> 今日の1つだけの行動を提案するAIコーチ。"**

### 詳細化

* **個別の生活パターンの発見**

  * 睡眠・HRV・活動量・会議・夜スマホのデータから、

    **「どんな日・どんな行動があなたの疲労や集中に効いているか」**を個人単位で推定。

* **日次の「1日1アクション」**

  * 毎朝、

    > 「今日は◯◯を1つだけ意識してください」

    > という **1つだけの具体的アクション**を提示（やらないことも含む）。

* **週次の"因果インサイト"**

  * 「夜のスマホが60分を超えた日は深い睡眠が平均14%低下しています」

  * 「会議が4件以上ある日の翌日はHRVが平均22%低下しています」

    など、**あなた専用の疲労・集中パターン**を可視化。

* **入力ほぼゼロ**

  * データはウェアラブル・カレンダー・アプリ内ログから自動取得。

  * ユーザーは日記を書かなくてよい。

## MVPの主要機能

### データ連携

| ソース                            | 取得する主なデータ（MVP）                                                                    | 何に使うか                     |
| ------------------------------ | --------------------------------------------------------------------------------- | ------------------------- |
| ウェアラブル（Oura / Apple Watch / 他） | 総睡眠時間 / 深い睡眠比率 / 入眠潜時 / 覚醒回数 / HRV / 安静時心拍 / 活動量・ステップ数 / readiness や activity スコア | 「回復状態」と「疲労の蓄積」のベース指標      |
| Googleカレンダー                    | 1日の予定数 / 会議数 / 会議時間合計 / 時間帯別の会議密度 / 終日の予定 / 出張などの重い予定                             | 仕事負荷・ストレス源の推定             |
| スマホ（アプリ内 or OSの Screen Time）   | 1日のスマホ利用時間 / 特に22:00–2:00の夜間スマホ時間                                                 | 睡眠を壊している行動の推定（最大の差別化ポイント） |
| Mac（Screen Time or RescueTime）        | 1日のPC利用時間 / 22:00–2:00の夜間PC作業時間 / 主要アプリの使用時間（Slack、メール、コーディングなど） / 時間帯別の作業パターン | PC作業パターンと疲労・睡眠の因果関係の分析（デスクワーカー向け） |

※ MVP 時点では「日次集計」でOK（リアルタイム不要）。

### アウトプット

#### ① 毎朝の「Today card」

* 表示タイミング：朝（起床後〜出社前）

* 含める情報：

| 要素           | 例                                             |
| ------------ | --------------------------------------------- |
| 今日のコンディション   | 「今日は**やや疲労高め**の日です。」                          |
| 簡易説明         | 「睡眠時間は7時間でしたが、深い睡眠がいつもより少なめ＋昨日の会議密度が高かった影響。」  |
| 今日の1アクション    | 「**15:00以降は単純作業に寄せて、重い意思決定は午前中に寄せてください。**」    |
| 避けるべきこと（あれば） | 「今日は**追加のカフェインは2杯まで**に抑えた方が、明日のHRV低下を防げそうです。」 |

#### ② 週次レポートの「Pattern Insight」

* 週1回、次のようなコピーを出す：

> **「先週のあなたのパターン」**

> ・夜スマホが60分を超えた日の深い睡眠：平均 14% 低下

> ・会議が4件以上ある翌日のHRV：平均 22% 低下

> ・平日に散歩が20分以上あった翌日のレディネス：平均 +9pt

> → **今週は「夜スマホ60分以内」「1日20分の散歩」を守るだけで、

> かなり回復しやすくなる可能性があります。**

## 競合サービス分析

### 主要競合サービス

#### 1. Lifestack
**URL**: https://lifestack.ai/

**概要**: AIアシスタントを活用して、タスク、食事、睡眠、集中時間をユーザーの実生活に合わせて計画するサービス。ウェアラブルデータとカレンダーを統合し、健康と仕事のバランスを最適化する。

**主な機能**:
* **Energy Zones**: ユーザーのピークエネルギーに合わせて重要な作業を配置し、生産性を最大化。カレンダー上にエネルギーレベルをオーバーレイ表示
* **Meals and Sleep**: AIがその日の要求に応じて食事と休息を計画。朝にバランスの取れたプランを提示
* **Wearable Insights**: 健康データをカレンダーと統合して表示。ウェアラブルからのデータを直接カレンダー上で確認可能
* **24/7 Copilot**: 日々の意思決定をサポートする常時稼働のAIアシスタント。コンテキストを理解し、適応的なサポートを提供
* **タスク管理**: Trelloなどのサードパーティプラットフォームと連携可能

**特徴**:
* iPhone、Android、Chromeで利用可能
* ウェアラブルデータとカレンダーを統合
* タスク管理と健康管理を一体化
* リアルタイムでの計画調整
* プライバシー重視（データは売却・共有されない）

**ユーザーボイス**:
* 「エネルギーレベルに基づいてタスクをスケジュールすることで、効果的になった」
* 「カレンダー上にエネルギーピークをオーバーレイできる唯一のアプリ」

### risely の独自性とLifestackとの具体的な違い

#### 1. **夜スマホと睡眠・疲労の因果関係の可視化（最大の差別化ポイント）**

**Lifestack**: 
* ウェアラブルデータとカレンダーを統合して表示
* エネルギーレベルに基づいたタスク配置
* スマホ利用データの分析は提供されていない

**risely**:
* **夜間（22:00-2:00）のスマホ利用時間を追跡**
* **スマホ利用時間と睡眠の質・HRVの因果関係を可視化**
* **週次レポートで「夜スマホが60分を超えた日は深い睡眠が平均14%低下」などの具体的な数値を提示**
* これが他のサービスでは提供されていない最大の差別化ポイント

#### 2. **1日1つだけの行動提案（シンプルさ）**

**Lifestack**:
* タスク、食事、睡眠、集中時間を包括的に計画
* 複数の要素を同時に最適化

**risely**:
* **毎朝「今日の1つだけの行動」を提案**
* 複数のアドバイスではなく、最も効果の高い1つの行動だけに絞る
* ユーザーの負担を最小限にし、継続しやすさを重視

#### 3. **段階的なパーソナライズ（学習の可視化）**

**Lifestack**:
* パーソナライズの期間や段階は明示されていない
* リアルタイムでの適応を重視

**risely**:
* **Day 0-3: ルールベース（汎用的なアドバイス）**
* **Day 4-7: 軽いパーソナライズ開始（仮説レベルの提示）**
* **Day 8-21: 個人モデルが改善（週次レポートでインサイト提供）**
* **Day 21-30以降: 本格的なパーソナライズ（生活パターンマップの完成）**
* 「学習して賢くなっていく感」自体が価値になる

#### 4. **バッチ処理による安定性（技術的アプローチ）**

**Lifestack**:
* リアルタイムでの計画調整を重視
* 24/7のコパイロットとして常時稼働

**risely**:
* **リアルタイム処理を避け、1日1-2回のバッチ処理でデータを取得・分析**
* システムの安定性と効率性を確保
* 日単位の粒度で十分な価値を提供

#### 5. **週次レポートでの因果インサイト（深い分析）**

**Lifestack**:
* カレンダー上に健康データを統合表示
* エネルギーレベルの可視化

**risely**:
* **週次レポートで因果インサイトを提供**
* 「夜のスマホが60分を超えた日は深い睡眠が平均14%低下」
* 「会議が4件以上ある日の翌日はHRVが平均22%低下」
* 「平日に散歩が20分以上あった翌日のレディネス：平均 +9pt」
* など、**個人専用の疲労・集中パターンを数値で可視化**

#### 6. **会議密度と生体データの相関分析**

**Lifestack**:
* カレンダーと健康データを統合表示
* エネルギーレベルに基づいたタスク配置

**risely**:
* **会議密度（1日の会議数、会議時間、時間帯別の密度）を詳細に分析**
* **会議密度とHRV・睡眠の質の相関を個人単位で特定**
* 「会議が4件以上ある日の翌日はHRVが平均22%低下」などの具体的な因果関係を提示

## キャリブレーション期間

> **最初の7日間は、一般的な睡眠・疲労パターンに基づいたアドバイスを行います。

> 14〜30日目以降は、あなた専用の"生活パターンモデル"が少しずつ完成していきます。**

> **risely は、使えば使うほど"あなたらしく"なっていきます。**
